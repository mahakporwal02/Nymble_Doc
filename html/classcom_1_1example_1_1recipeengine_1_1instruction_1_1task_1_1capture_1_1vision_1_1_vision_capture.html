Detailed Description

This class implements CapturableTask and contains the properties and methods to capture either RGB or thermal images by running the algorithm in separate threads.

◆ beginTask()

synchronized void com.example.recipeengine.instruction.task.capture.vision.VisionCapture.beginTask ( )

This function is responsible for running the CaptureTask of a specific VisualCaptureType.

  1. First the function checks if there is a trigger event given in the sensor params or not.
  2. If there is no trigger event the execution shall continue as usual.
  3. If it is present then the we wait for Julia State to change from idle to running we fire a propertyChangeEvent to change begin from false to true. The timeout is set for 120000 ms.
  4. Then we use the VisionTaskFactory to get the correct Thread implementation based on VisualCaptureType present in sensorParams
  5. Then we run the start method of the thread object.
